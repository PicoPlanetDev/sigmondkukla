Dashboard Actions on Google

This Google Assistant Action allows you to ask Google Assitstant about your grades. It retrieves grade information from Powerschool SIS and displays it as part of a conversation with Google Assistant.

The check_grade intent is triggered and the user query is parsed to determine what class to retrieve from my endpoint. The class is a custom type with class names that appear in Powerschool SIS. Each name has synonyms that I use to refer to my classes. This calls my webhook which gets the grade information from Powerschool and returns it to Google in a friendly format.

I used Glitch to host this project for developmet and testing. Because my current goals for this project remain somewhat limited, I haven't implemented many features yet, such as using your own login information.

On my first attempt to host my webhook endpoint, I used PythonAnywhere. However, the endpoint must make requests to a Powerschool URL that is not on the PythonAnywhere whitelist. This brings up another problem in that Glitch makes apps sleep after five minutes and have a slow startup. Often, the webhook request will time out in Actions on Google. To combat this, I added a wake handler to my endpoint that (as quickly as possible) returns a ready response.

Through building and deploying this, I learned:

  • Using Google Actions Console
  • Creating a Flask application
  • Parsing JSON
  • Hosting a Python application
  • Using Glitch
  • Using Postman to test requests
